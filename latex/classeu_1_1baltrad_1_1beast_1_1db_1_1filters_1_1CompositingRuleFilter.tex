\doxysubsection{Detailed Description}
A filter for collecting and matching files for a specific compositing rule, within a time period. 

The filter can either be setup to be scan-\/based or volume-\/based, just as the compositing rule. If scan-\/based, it will match files with \char`\"{}what/object\char`\"{} set to \char`\"{}\+SCAN\char`\"{}. Otherwise it will match files with \char`\"{}what/object\char`\"{} set to \char`\"{}\+PVOL\char`\"{}.

The filter can both be used to check if specific files match the filter (by using method \mbox{\hyperlink{classeu_1_1baltrad_1_1beast_1_1db_1_1filters_1_1CompositingRuleFilter_a93caeb5f87240ee1b37bfc6060708980}{file\+Matches(\+File\+Entry)}}) and for getting a File\+Query to use towards the database, to get all files matching the parameters of the filter.

When the filter is use to query the database, the returned files will be sorted on time (\char`\"{}what/date\char`\"{} + \char`\"{}what/time\char`\"{}), with the oldest file first.

If the filter is set to be scan-\/based, the files will also be sorted on angle, ~\newline
 with the lowest angle (\char`\"{}where/elangle\char`\"{}) first, when applied in a query for the database, if there are several matches for the other parameters of the filter.

The filter can also be provided with a list of sources. The filter will then only match/return files that that have a source id matching any of the sources in the list.

It is also possible to define a start -\/ stop date/time interval. If so, it will only fetch files within this time period, if used for querying the database. You can also ignore specifying start and/or stop date/time but if you ignore both you probably want to use a different filter.
